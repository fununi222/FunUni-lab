Freeman Project HP

情報設計と見やすさを意識したホームページ制作のふりかえり

// 過去制作したホームページを、設計意図と学びの観点から振り返る記録

過去に制作した Freeman Project HP は、情報を整理してわかりやすく届けることを目的に作ったホームページです。今振り返ると、派手な機能を増やすよりも、「何を、どの順番で、どう見せるか」 を強く意識していた制作だったと感じます。

1. 制作背景

当時は、単にページを作るだけではなく、訪れた人が迷わず内容を理解できるホームページを目指していました。ホームページは一見シンプルですが、情報量、見出し、余白、色のトーンといった細かい要素の積み重ねで印象が大きく変わります。

この制作では、デザインと実装を切り分けて考えるのではなく、見た目の印象と読みやすさを両立させること をテーマに進めました。

2. 制作で意識したこと

情報が自然に流れる構成

訪問者が最初にどこを見て、次に何を知りたくなるのかを意識しながら、ページ全体の流れを組み立てました。情報を詰め込みすぎると読みにくくなるため、見出しやセクションの役割を整理し、必要な内容が自然に目に入る構成を目指しました。

見た目より先に可読性を整える

ホームページ制作では、装飾を増やせばよく見えるわけではありません。そこで今回は、配色やアニメーションより先に、文字サイズ、余白、要素間のまとまり方を優先して調整しました。結果として、派手すぎず、それでも印象に残るページに近づけたと思います。

レスポンシブ対応

PC だけでなくスマートフォンでも自然に見えることを意識して調整しました。特に、画面幅が変わったときの余白や要素の並びは印象に直結するため、崩れないことだけでなく、小さい画面でも読みやすいこと を重視しました。

3. 制作を通して学んだこと

このホームページ制作を通して強く感じたのは、Web 制作では「作れること」と「伝わること」は別だという点です。HTML や CSS を使って形にするだけならある程度進められても、相手にとって見やすく、理解しやすい形に整えるには、もう一段深い視点が必要でした。

また、完成直後には気づけなかった改善点も、時間が経ってから見返すことで見えてきました。コードの整理、再利用しやすい構成、命名の統一など、今ならもっと保守しやすい形にできると思います。

4. 今ならこう改善する

  1. コンポーネント単位で整理し、保守しやすい構成にする
  2. 余白やタイポグラフィをさらに設計し、ブランド感を強める
  3. ページ導線を改善し、ユーザーが次の情報へ進みやすくする

過去の制作物を振り返ると、その時点での技術力だけでなく、どこまで相手目線で考えられていたかも見えてきます。Freeman Project HP は、自分にとって 「デザインと情報設計を同時に考える大切さ」 を学べた制作のひとつでした。
